1. Хранение уникальных идентификаторов пользователей
В системе есть список уникальных идентификаторов пользователей.
Необходимо быстро проверять, присутствует ли пользователь с заданным идентификатором.

Решение: Подойдет потому что Set хранит уникальные значение и обеспечивает быстрый поиск по значению


2. Очередь задач для обработки
Система должна обрабатывать запросы в порядке их поступления. Требуется поддерживать последовательность.

Решение: Необходимо использовать очередь, тк она работает по принципу FIFO - первым зашел - первым вышел

3. Хранение ключей и значений
Необходимо хранить настройки приложения, где каждый параметр имеет имя (ключ) и значение.

Решение: Данные хранятся парой ключ-значение, подойдет структура Map и она обеспечивает быстрый поиск по ключу

4. Иерархия сотрудников компании
Требуется сохранить данные о сотрудниках в виде дерева: каждый сотрудник имеет подчинённых.

Решение: Подойдет структура Дерево, так как оно хранит иерархические данные и поддерживает отношения
родитель-потомок

5. Поиск элементов по приоритету
Система обрабатывает задачи, причём приоритет задачи определяет, когда она будет выполнена.

Решение: Подойдет структура данных Приоритетная очередь, так как обрабатывает данные в порядке приоритета


6. Управление друзьями пользователя
Требуется хранить связи между пользователями в социальной сети,
где каждый пользователь может быть связан с несколькими другими.

Решение: Подойдет структура данных граф, так как она использутся для моделирования связей между объектами

7. Упорядоченный список
Необходимо хранить список чисел в отсортированном виде, чтобы быстро находить минимальное и максимальное значения.

Решение: Использовать структуру данных дерево - дерево BST, оно поддерживает порядок элементов и позволяет быстро получить
минимум и максимум


8. Учет посещений страниц
Система должна отслеживать уникальные посещения страниц на веб-сайте и вести подсчёт общего числа посещений.

Подсказка: выбор структуры данных может быть разный в зависимости от уточнения,
уникальные ИЛИ не уникальные общие посещения мы считаем.

Решение: Если нужно хранить уникальное значение, то нужно использовать множество Set

9. История действий пользователя
Необходимо хранить список последних действий пользователя, чтобы можно было отменить или повторить действия.

Решение: Необходимо использовать структуру данных стек, так он работает по принципу LIFO обрабатывает последний
принятый элемент

10. Структура телефонной книги
Требуется хранить имена и номера телефонов, чтобы быстро находить номер по имени.

Решение: Необходимо использовать Map тк данные хранятся в паре ключ-значение, Map позволяет быстро найти
значение по ключу

11. Проверка корректности скобочной последовательности
Пользователь вводит строку, содержащую скобки. Нужно определить, правильно ли расставлены скобки.

Решение: Подойдет стек, потому что алгоритм такой:
- найти и сохранить открывающую скобку
- если встречается закрывающая скобка подходящего типа, то открывающая скобка удаляется
если в конце стек пустой, то скобочная последовательность правильная, в ином случаем скобочная последовательность
некорректная


12. Хранение объектов с частым доступом
Требуется хранить фиксированное количество элементов, которые часто изменяются, с быстрым доступом к любому из них.

Решение: Подойдет массив, так как он имеет фиксированное значение и обсепечивает быстрый доступ к данным по индексу

13. Уникальные слова в тексте
Дан текст, нужно сохранить все уникальные слова и определить их количество.

Решение: Нужно использовать множество, так как оно хранит только уникальные значение

14. Структура электронной очереди
В поликлинике пациенты записываются в очередь, и каждый должен быть вызван в порядке записи.

Решение: Нужно использовать очередь, так как она работает по принципу FIFO и обрабатывает данные
в порядке поступления

15. Список дел
Требуется сохранить список задач, который можно пополнять, удалять и сортировать по сроку выполнения.

Решение: Подойдет список потому что обеспечивает быстрое добавления данных

16. Анализ частоты слов
Требуется определить, сколько раз каждое слово встречается в тексте.

Решение: Подойдет словарь, ключ - это слово, а значение - количество повторов

17. Система рейтингов
Вам нужно хранить список игроков с их рейтингами, чтобы быстро находить игрока с самым высоким рейтингом.

Решение: Подойдет приоритетная очередь, потому что позволяет получить элемент с максимальным рейтингом

18. Проверка дубликатов в потоке данных
Данные поступают в потоковом режиме, и нужно проверять, был ли данный элемент уже получен ранее.

Решение: Подойдет множество, так хранит только уникальные значения и автоматически проверяет их наличие
